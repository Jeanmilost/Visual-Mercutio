Configuration des logiciels Monitor

 

Généralités

 

Etant donné que Monitor est composé de plusieurs logiciels différents, certains points de configuration doivent être effectués. Monitor gère également plusieurs univers (base de données) différents. Il faut donc les configurer pour qu’ils soient reconnus à travers les différents outils de la suite Monitor.

Monitor est composé d’un ensemble de logiciels utilisant différents environnements.

 

Webservice emessenger (publication depuis Conceptor)

Lors de la publication d’un modèle depuis Conceptor, le service web à besoin de connaître l’univers concerné par cette nouvelle publication. Un fichier alias.ini permet de lister les différents nom d’univers ainsi que leur correspondance au niveau de la base de données. En résumé un univers pointe vers une base de données.

 

Le fichier se trouve sur c:\Program Files\Apache Group\jakarta-tomcat-4.0.1\bin\alias.ini

-          1 ligne par alias

-          1 retour à la ligne après la dernière ligne d’alias

-          Structure : [nom d’alias]=[lien sur base de données physique] (exemple : simulation=jdbc:mysql://localhost:3306/simulation)

 

Le fichier par défaut contient déjà 3 alias pointant sur les trois bases de données fournies.

 

BatchMessages

Le répertoire de l’outil BatchMessages contient différents fichiers nécessaires à l’exécution du programme. Des fichiers .bat préconfigurés permettent de lancer l’exécution du programme java.

-          ExecuteBatchMessages-pilotage.bat  (exécute en travaillant sur l’univers pilotage)

-          ExecuteBatchMessages-workflow.bat  (exécute en travaillant sur l’univers workflow)

-          ExecuteBatchMessages-simulation.bat  (exécute en travaillant sur l’univers simulation)

 

Si vous créez de nouveaux univers il faudra créer un nouveau fichier .bat (en se basant sur le contenu des 3 existants). Le seul changement à effectuer se trouve sur la dernière ligne du batch (avant la pause). Le dernier paramètre passé est le nom de l’univers concerné. (Univers défini dans le fichier alias.ini)

 

Team Editor

Au lancement de Team Editor, une fenêtre liste tous les univers existant et il faut en choisir un. Il faut donc configurer un fichier pour les définir. Par défaut, les trois univers (pilotage, workflow, simulation) sont déjà définis. Pour modifier cette liste il faut éditer le fichier TeamEditor.exe.config se trouvant dans le répertoire TeamEditor de votre installation Monitor.

 

- repérer les éléments <add key=’’xxxxxxx’’….

-        L’élément <add key="Processsoft.ConnectionString" …. Ne doit pas être modifié

-        Les élements suivants “add …” peuvent être modifiés / supprimés / ajoutés en fonction des univers devant exister.

-        L’attribut « key » correspond au nom de l’alias (idem à ceux créés dans alias.ini pour java)

-        L’attribut « value » doit comprendre la chaîne de connexion physique à la base de données :

à         Database=workflow;Data Source=localhost;User Id=root;Password=root

à         Ici Database=workflow correspond au nom de la base “réelle” dans mysql
