Publication d’un modèle depuis Conceptor

 

Généralités

Cette étape est nécessaire car c’est celle qui permet d’alimenter la structure de la base de données qui sera utilisée par Monitor. Une fois le modèle terminé selon la méthode Mercutio à l’aide de l’outil Conceptor, il faut utiliser le menu « fichiers - publier le modèle vers messenger» pour obtenir l’écran de publication

 

 

L’adresse correspondant à l’url du service web. Si le serveur tomcat se trouve ailleurs que sur localhost il faut changer l’adresse pour indiquer quel serveur utiliser.

 

L’alias doit correspondre à un alias existant dans le fichier alias.ini. Celui-ci indiquera quelle est la base de données qu’il faut utiliser.

 

En cliquant sur le bouton suivant, l’écran des options sera affiché :

 

 

La date de validité correspond à la date de début de validité du modèle. Selon la date saisie, les modèles précédents seront automatiquement désactivés.

 

L’utilisateur et le mot de passe sont par défaut : admin / toto. Il s’agit d’un utilisateur avec les droits admin définis dans la table « pemployee »

 

Le bouton « publier » permet de lancer le traitement.

 

Ne pas oublier de lancer le serveur tomcat avant la publication !
