XML Builder et Composition des messages

 

Généralités

Monitor possède un service Web qui permet de recevoir un ou plusieurs messages d’un système externe.

L’url du service web est : http://[SERVER]:8080/soap/servlet/rpcrouter

 

Structure du fichier XML:

 

<root>

        <message caseid="" caseidremark="" externmessageid="" externprocid="" externsystemid="" externuserid=" " messagetime="" statut="" timestampproc="" externprestationid="">

<messagedeliv doublecontrol="" externdelivid="" externdelividint="" externuserid=" " timestampdeliv="" />

  </message>

  </root>

 

-          caseid : numéro de cas

-          caseidremark : facultatif, chaîne de caractères à remarque concernant le cas

-          externmessageid : numéro du message du système externe – facultatif

-          externprocid : identifiant de la procédure sur le système externe

-          externusid : identifiant de l’utilisateur sur le système externe

-          messagetime : date/heure du message sous le format : « yyyymmjj hhmmss »

-          statut : 0 ou 1, correspond à un livrable final ou non.

-          timestampproc : date/heure de validation de la procédure sur le système externe, format « yyyymmjj hhmmss »

-          externprestationid : numérique, identifiant de la prestation

-          doublecontrol : 0 ou 1, indique si le double contrôle à lieu ou non

-          externdelivid : facultatitf, identifiant unique du livrable

-          externdelividint : obligatoire : identifiant du livrable du système externe

-          externuserid : identifiant de l’utilisateur du système externe

-          timestampdeliv : date/heure de validation du livrable sous le format : « yyyymmjj hhmmss »

 

 

Convertisseur Excel - > XML

Etant donné que toutes les entreprises n’ont pas la possibilité d’automatiser l’envoi de message en utilisant directement le service web de Monitor, un outil de conversion de fichier Excel vers XML est existant.

Cet outil à pour but de convertir un fichier Excel (sous un format défini) en fichier XML pouvant être interprété par le service Web de Monitor.

 

Le convertisseur va s’appuyer sur deux critères pour générer un nouvel identifiant selon la sélection de l’utilisateur

 

La structure du fichier Excel provenant par exemple d’une exportation de données depuis l’entreprise est définie comme cela :

 

Colonne A : libre

Colonne B : critère 1

Colonne C : critère 2

Colonne D : date et heure du message (03.01.2006 16:05:42)

Colonne E : ID de l’utilisateur

Colonne F : Numéro de prestation

Colonne G : Numéro du tiers (cas)

Colonne H : Statut

Colonne I : identifiant du système externe, 1 par défaut

Colonne J : identifiant du Message

 

 

Etant donné que sur un système externe il n’est parfois pas possible d’identifier une procédure par un seul critère, l’outil de conversion permet de créer un identifiant à partir de deux critères correspondants aux colonnes du fichier excel.

 

Dans l’exemple ci-dessus, si une ligne du fichier excel contient « Valider Réponse » dans la colonne du critère 1, et « Office 2 » en critère 2, le fichier XML généré aura comme identifiant de procédure 4, et livrable 1. Ainsi la lecture de la ligne dans le fichier excel indique que le livrable 1 à été créé et la procédure 4 terminée.

 

Le bouton « sauver » permet de sauvegarder le tableau des critères

Le bouton « charger » permet de charger le tableau des critères.

 

Avant de générer le fichier XML, il faut sélectionner le fichier Excel source, sélectionner un fichier XML de destination.

 

Le bouton «  génération » permet de lancer la procédure de conversion.
